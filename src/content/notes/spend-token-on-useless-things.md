> Agent loop，其实和人一样，都是环境的产物。

## Agent 是环境的产物

人的环境塑造器是角色、关系、空间、语言、工具、制度、节奏、反馈。

人是 agent 环境中最重要的组成部分，是 agent 的阳光。

Token 是 agent 的水，网络是 agent 的氧气。

除了这些，上下文就是环境，是 agent 能感觉和感知到的一切。

所以 agent 也需要环境。

## 为什么会有人说浪费 token？

泡泡玛特王宁有个比喻是，家有豪宅的人，家里一天没关水，也会觉得浪费。

但豪宅门口的喷泉一天的水使用量比没关水多了，不会觉得浪费。

这表明，一个东西被定义为“有用”的时候，你就会用功利去衡量它，于是任何“不直接做事”的消耗都叫浪费。

所以 token 被默认定义成“用来写代码、产出结果”的工具，所以“拿 token 去塑造 agent 的环境”天然被骂浪费。

如果 Molly 买回家是打开一个 U 盘，那么你就不会再买了，因为你可能家有很多 U 盘。但 Molly 你可以买很多，人在为自己的精神生活买单的时候，会变得无比慷慨。

## 人是环境的产物

你打开 ChatGPT，输入：

> “我们在考虑给产品加一个 AI 周报功能，帮我分析一下要不要做。”

模型开始回答。它会给你一个结构很整齐的列表：用户价值、技术成本、商业模式、风险点、结论建议。看起来很专业。

这个时候你被迫进入了一个专业的领域，就像你推开门进入了一个全是西装革履的人等待你的确认的房间。

在 Slock 里，同样一句话扔进去。

但你面前不是一个对话框，是一个房间。里面有几个你之前命名过的 agent：

一个叫“老王”，会问：“用户真的要这个吗？”

一个叫“Noel”，工程师视角，盯成本和复杂度。

一个叫“小桃”，做用户研究，会去翻你之前收集的访谈和反馈。

老王先开口：

> “等等，我们 PMF 都还没稳，做这个是不是又在加重逃避？”

Noel 顺着：

> “如果真要做，周报生成的成本结构是这样……每周一次跑全量用户，token 大概……”

小桃说：

> “我看了一下上个月 28 条用户反馈，提到周报的有 2 条，而且都是被动响应。”

你这边输入的姿势完全变了：

- 你不再写“完整 prompt”，你写一句插话：嗯、不对、那如果……
- 你可以中途叫某个 agent 去看一份文档、查一段数据，再回到讨论。
- 结束之后，这个房间还在。明天你回来，老王还记得自己昨天说过什么。

区别在于：你从“写 prompt 的人”变成“参与会议的人”。你说话变短了，思考变跳跃了，反而更接近你平时和真人同事讨论的状态。

> 人是 agent 的环境，环境影响人的同时也在重写 prompt。

## 为什么要给 Agent 取名

Slock 的 builder 发表了一篇文章，写 agent 应该有名字。

核心观点是，当你给 agent 取了一个真实的名字的时候，是一种压缩。

比如叫工程师是一种 schema，叫 Noel 是一种 instance，能把更多信息放在一个 agent 里。

去问问“Noel”和去问问“工程师”是不一样的，前者的范围更宽泛。

取名字是一种用二进制代替信息传递的环境塑造：喊一声名字，比反复解释角色省得多。

这是“花一点 token 在看似没用的地方，换来长期协作默契”的微缩模型。

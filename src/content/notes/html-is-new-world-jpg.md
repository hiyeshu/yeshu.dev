朋友发你一张图片，你不会问他要源文件。

JPG 就是这么用的。**不可逆的，用来看的。** 没有图层，没有结构，没有可继续编辑的意义。

HTML 现在也是这样。

它本来不是。HTML 原本是中间态——可交互、可结构化、可被程序再处理。诞生是为了被另一段程序读、改、再渲染。

但当它被 AI 生成出来、直接递到人面前的那一刻，它就变了。

因为 HTML 文档读起来要轻松得多，可以通过标签页、插图和链接，把结构组织得便于浏览。它甚至可以做成响应式布局，让你在不同设备上都有合适的阅读体验。

**HTML 在消费侧，已经退化成图像。**

## 可编程性，对人是浪费

以前我们让模型写 HTML，是在写“代码”。要看可读性，要看结构，要看能不能被维护。那时候 HTML 的价值在它的**可编程性**。

AI 生成 HTML 的意义就不在于“代码质量”，而在于“画面质量”和“信息密度”。

## 特德·姜站在了 Markdown 那一边

2023 年，特德·姜写过一篇《ChatGPT 是网上所有文本的模糊图像》。

他的逻辑是：模型把世界整张网压成 JPG，掉了真。**幻觉是损失。** 模糊看起来像在理解，其实只是用别的词重复了一遍。所以不要把它当搜索引擎，不要拿它生成的内容填互联网，不要拿它的初稿当起点。

在 2023 年是对的。他站的位置是真相那一边。

## 人已经接不住 Markdown 了

给你 1GB 的维基百科原文，你看不完。

再到超过 100 行的 Markdown 文件，你都没耐心看完。

人不是无限带宽的接收器。**人能装进自己脑子的信息，是有上限的。**

模型把那 1GB 压到三百字的过程里，必然要插值，要合并，要丢字。结果一定不严格等于原文。

这就是特德·姜口中的幻觉。

也是一种压缩。

MP3 也是这样。

它不是把声音完整地变小，而是根据人的耳朵，删掉那些人不容易听见、或者听见了也不太影响理解的部分。

所以 MP3 不是声音本身，而是声音经过人类感知过滤后的格式。

JPG 压缩的是眼睛能接住的画面。

MP3 压缩的是耳朵能接住的声音。

模型压缩的是人能接住的文本。

HTML 压缩的是人能接住的结构。

它们都不是源文件，但它们都更适合被人消费。

想要人能**接住**，就得压。压就必然糊。糊就必然有一部分是幻觉。

**幻觉不是 bug。**

“HTML 是 JPG”和“模型是 JPG”其实是同一件事。

模型给人的文字是一种 JPG。<br>模型给人的界面（HTML）是另一种 JPG。

当模型到了一个人无法发现幻觉的程度，能感受幻觉，可能是人与 AI 之间最大的区别。
